# 2077日报 — 产品设计文档

> 来自2077年的疯狂新闻社区
> "疯子写的 · 先知读的"

## 1. 产品定位

**一句话描述：** 一个以"来自2077年的新闻报道"为形式，让用户分享疯狂未来预测的社区网站。

**核心理念：** 受 SHIT期刊 启发，用最正经的新闻格式包装最疯狂的未来设想。形式反差产生幽默感和传播力。

**产品调性：** 不是严肃的未来学平台，而是一份"疯子报纸" — 用户以"2077年记者"的身份，用新闻报道的格式，写出他们对未来最疯狂的想象。

## 2. 目标用户

不预设用户画像，让产品自然吸引受众。潜在人群包括：
- 科技极客（在脑洞中夹带真实技术推演）
- 创意青年（寻找情绪出口和创作乐趣）
- 科幻爱好者（享受世界观构建）

## 3. 核心机制

### 3.1 模板化投稿

用户选择模板，填写核心创意（一句话），Ark 自动扩写为完整的2077风格新闻报道。

| 模板 | 用户输入 | Ark 生成 |
|------|---------|---------|
| 📰 头条新闻 | 核心事件 + 时间 + 地点 | 标题、副标题、导语、正文3段、记者署名 |
| ⚡ 突发快讯 | 发生了什么（一句话）| 快讯标题、3句话概要、持续关注提示 |
| 🪦 讣告/终结公告 | 什么东西"死了"（如现金、996...）| 正式讣告格式：生卒年、生平、悼词 |
| 📢 2077广告 | 产品/服务名称 + 描述 | 广告文案、卖点、价格（2077货币）、免责声明 |

### 3.2 赛博职业身份

用户注册时随机分配一个2077年的职业身份，显示在所有发帖和评论的署名上。

预置50+职业，例如：
- 量子外卖员
- 火星房产中介
- AI心理咨询师
- 意识云存储管理员
- 基因编辑美容师
- 时空快递调度员
- 虚拟宠物兽医
- 数字货币考古学家

### 3.3 双维投票

每条新闻接受两个维度的社区投票（1-5分）：

- **🔥 疯狂指数** — "这想法有多疯？"（1=正常，5=疯透了）
- **🎯 成真概率** — "这事可能真发生吗？"（1=做梦，5=板上钉钉）

最有趣的新闻 = 疯狂指数高 × 成真概率也高（"太疯了但好像真的会发生"）。

### 3.4 分享海报

发布新闻后自动生成报纸风格海报，支持：
- 竖版（适配朋友圈/小红书）
- 横版（适配微博/X）
- 一键保存图片 / 分享链接

海报是传播的核心引擎，设计需要足够有冲击力让人想转发。

## 4. 交互机制（分阶段上线）

### Phase 2 — 社区沉淀
- 新闻流排序（热门/最新/最疯狂/最可能成真）
- 排行榜
- 角色扮演评论（用户以2077市民身份留言）

### Phase 3 — 深度互动
- 协作续写（其他用户为新闻写"后续报道"，形成故事链）
- 栏目分类（科技/社会/太空/AI/生物...）
- 先知认证（预测成真时回溯验证）

## 5. 视觉风格

**"信号截获"风格：**
- 纯黑底色
- 霓虹双色系：红(#ff0050) + 青(#00f0ff)
- 像截获了来自未来的信号
- 标签样式：`[头条]` 红色，`[科技]` 青色
- 顶部扫描线动画
- 报头：`DISPATCH FROM THE FUTURE`
- 副标题：`SIGNAL DETECTED · YEAR 2077 · FREQUENCY 未知`

网站整体为暗色调，信息以"信号流"的形式呈现，带有轻微的故障艺术(glitch)效果。

## 6. 页面结构（Phase 1）

### 6.1 首页 — 信号接收站
- 报纸/信号风格排版的新闻流
- 按 热门/最新/最疯狂/最可能成真 排序
- 每条新闻显示：标题、导语、作者职业、投票数
- 无需登录即可浏览

### 6.2 发布页 — 信号发射台
- 选择模板（头条/快讯/讣告/广告）
- 填写核心创意（一句话）
- Ark 自动扩展为完整2077风格报道
- 预览 → 发布 → 生成分享海报

### 6.3 详情页 — 信号解析
- 新闻全文展示
- 双维投票按钮
- 分享海报生成按钮
- SEO友好，独立URL

### 6.4 个人页 — 记者档案
- 赛博职业名片
- 已发布的新闻列表
- 累计投票统计
- Supabase Auth 登录（GitHub/Google/邮箱）

## 7. 技术架构

```
Next.js 15 (App Router)  ←→  Supabase  ←→  Ark API
       ↓                        ↓
    Vercel 部署            PostgreSQL
                           Auth 认证
                           Storage 存储
```

| 层 | 技术选型 |
|----|---------|
| 框架 | Next.js 15 (App Router, React Server Components) |
| 部署 | Vercel |
| 数据库 | Supabase PostgreSQL |
| 认证 | Supabase Auth |
| 样式 | Tailwind CSS |
| AI生成 | 火山引擎 Ark API |
| 海报渲染 | Satori (Vercel OG) — 边缘渲染 |
| 备用环境 | Cloudbase（后续扩展） |

## 8. 数据模型

### profiles
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | FK → auth.users |
| display_name | text | 显示名称 |
| cyber_job | text | 赛博职业名称 |
| avatar_url | text | 头像URL |
| created_at | timestamptz | 创建时间 |

### articles
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| author_id | uuid | FK → profiles |
| template | text | headline / flash / obituary / ad |
| user_input | text | 用户原始创意 |
| title | text | 新闻标题 |
| subtitle | text | 副标题 |
| content | text | Ark 生成的正文 |
| news_date | text | 2077年的新闻日期 |
| created_at | timestamptz | 创建时间 |

### votes
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| article_id | uuid | FK → articles |
| user_id | uuid | FK → profiles |
| crazy_score | int2 | 疯狂指数 1-5 |
| real_score | int2 | 成真概率 1-5 |
| created_at | timestamptz | 创建时间 |
| | UNIQUE | (article_id, user_id) |

### cyber_jobs
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| title | text | 职业名称 |
| department | text | 所属部门 |
| description | text | 职业描述 |

预置50+职业记录，注册时随机分配。

## 9. 分阶段交付

| 阶段 | 时间 | 内容 |
|------|------|------|
| Phase 1 | 2周 | 模板投稿 + AI生成 + 分享海报 + 赛博职业 + 新闻流 + 投票 |
| Phase 2 | +2周 | 排行榜 + 角色扮演评论 + 新闻分类 |
| Phase 3 | +3周 | 协作续写 + 先知认证 + 高级搜索 |

## 10. 成功指标

- **传播力：** 分享海报的社交媒体曝光量
- **创作量：** 每日新增新闻数
- **参与度：** 投票率（投票数/浏览数）
- **留存：** 7日回访率
